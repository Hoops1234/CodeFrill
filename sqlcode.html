SQL

What Is SQL?

SQL stands for Structured Query Language. It's the standard language for managing and manipulating relational databases.
SQL allows you to create, read, update, and delete data stored in a database.
Think of SQL like a translator between you and the database - you give commands in SQL, and the database understands exactly what to do.

queries, updates, inserts, deletes, and more.

How SQL Works

SQL uses statements (commands) to perform operations on a database. These statements are written in a specific syntax:
Most SQL statements follow this pattern:

  • VERB - what action to take (SELECT, INSERT, UPDATE, DELETE)
  • OBJECT - what to act on (table, column names)
  • CLAUSE - additional conditions (WHERE, ORDER BY, GROUP BY)

Everything follows a clear, readable structure.



SELECT Statement

The SELECT statement retrieves data from one or more tables.

SELECT column1, column2 FROM table_name;

Use SELECT * to retrieve all columns.

SELECT DISTINCT

Returns only distinct (different) values, eliminating duplicates.

SELECT DISTINCT column FROM table_name;

WHERE Clause

Filters records based on specific conditions.

SELECT * FROM table_name WHERE condition;

Example: WHERE age > 18

ORDER BY Clause

Sorts the result set in ascending (ASC) or descending (DESC) order.

SELECT * FROM table_name ORDER BY column ASC;

INSERT INTO Statement

Adds new rows (records) to a table.

INSERT INTO table_name (column1, column2) VALUES (value1, value2);

UPDATE Statement

Modifies existing records in a table.

UPDATE table_name SET column1 = value1 WHERE condition;

Important: Always use WHERE with UPDATE to avoid changing all rows!

DELETE Statement

Removes records from a table.

DELETE FROM table_name WHERE condition;

Important: Always use WHERE with DELETE to avoid removing all rows!

LIMIT Clause

Specifies the number of records to return (useful for large datasets).

SELECT * FROM table_name LIMIT 10;

MIN and MAX Functions

Returns the smallest (MIN) or largest (MAX) value in a column.

SELECT MIN(column) FROM table_name;

SELECT MAX(column) FROM table_name;

COUNT Function

Returns the number of rows that match a condition.

SELECT COUNT(column) FROM table_name;

SELECT COUNT(*) FROM table_name; counts all rows

AVG Function

Returns the average value of a numeric column.

SELECT AVG(column) FROM table_name;

SUM Function

Returns the total sum of a numeric column.

SELECT SUM(column) FROM table_name;

LIKE Operator

Searches for a specified pattern in a column (wildcard searches).

SELECT * FROM table_name WHERE column LIKE 'pattern';

  • % - represents zero or more characters
  • _ - represents a single character

Examples: LIKE 'A%' (starts with A), LIKE '%@gmail.com' (ends with @gmail.com)

IN Operator

Allows you to specify multiple values in a WHERE clause.

SELECT * FROM table_name WHERE column IN (value1, value2, value3);

BETWEEN Operator

Selects values within a given range (inclusive).

SELECT * FROM table_name WHERE column BETWEEN value1 AND value2;

AND, OR, NOT Operators

Combine multiple conditions in a WHERE clause:

  • AND - all conditions must be true
  • OR - at least one condition must be true
  • NOT - negates a condition

GROUP BY Statement

Groups rows that have the same values into summary rows.

SELECT column, COUNT(*) FROM table_name GROUP BY column;

HAVING Clause

Filters groups after GROUP BY (similar to WHERE but for aggregated data).

SELECT column, COUNT(*) FROM table_name GROUP BY column HAVING COUNT(*) > 5;

JOIN Operations

Combines rows from two or more tables based on a related column:

  • INNER JOIN - returns only matching records from both tables
  • LEFT JOIN - returns all records from left table + matching from right
  • RIGHT JOIN - returns all records from right table + matching from left
  • FULL JOIN - returns all records when there's a match in either table

Example: SELECT * FROM table1 INNER JOIN table2 ON table1.id = table2.id;

UNION Operator

Combines the result set of two or more SELECT statements (removes duplicates).

SELECT column FROM table1 UNION SELECT column FROM table2;

UNIQUE Constraint

Ensures all values in a column are different (no duplicates).

Primary Key

A column (or set of columns) that uniquely identifies each row in a table. Cannot be NULL and must be unique.

Foreign Key

A column that references the primary key of another table, creating a relationship between tables.

CREATE TABLE Statement

Creates a new table in the database with defined columns and constraints.

CREATE TABLE table_name (
   id INT PRIMARY KEY,
   name VARCHAR(100) NOT NULL,
   email VARCHAR(255) UNIQUE
);

Data Types

Common SQL data types:

  • INT / INTEGER - whole numbers
  • VARCHAR(n) - variable-length string (max n characters)
  • TEXT - long text strings
  • DATE - date (YYYY-MM-DD)
  • TIMESTAMP - date and time
  • DECIMAL(p,s) - exact numeric with precision p and scale s
  • BOOLEAN - true/false values
  • BLOB - binary data (images, files)

ALTER TABLE Statement

Modifies an existing table structure.

  • ADD column - adds a new column
  • DROP COLUMN - removes a column
  • MODIFY COLUMN - changes column data type

DROP TABLE Statement

Deletes an entire table from the database (all data + structure).

DROP TABLE table_name;

TRUNCATE TABLE Statement

Removes all rows from a table but keeps the table structure intact (faster than DELETE).

TRUNCATE TABLE table_name;

Indexes

Database objects that speed up data retrieval operations on a table column.

CREATE INDEX index_name ON table_name (column);

NULL Values

NULL represents missing or unknown data. It is NOT the same as zero or empty string.

  • IS NULL - checks for NULL values
  • IS NOT NULL - checks for non-NULL values

SQL Sub-languages (Categories)

DDL - Data Definition Language

Commands that define and modify database structure:

  • CREATE - creates new tables/database objects
  • ALTER - modifies existing structure
  • DROP - deletes tables/database objects
  • TRUNCATE - removes all rows from a table

DML - Data Manipulation Language

Commands used to manage data within tables:

  • SELECT - retrieves data
  • INSERT - adds new data
  • UPDATE - modifies existing data
  • DELETE - removes data

DCL - Data Control Language

Commands that control access to data:

  • GRANT - gives user access privileges
  • REVOKE - removes user access privileges

TCL - Transaction Control Language

Commands that manage transactions (groups of operations as single unit):

  • BEGIN / COMMIT - starts/ends a transaction
  • ROLLBACK - undoes changes in a transaction
  • SAVEPOINT - creates a point to roll back to

Best Practices

  • Always end SQL statements with a semicolon (;)
  • Use uppercase for SQL keywords (SELECT, FROM, WHERE)
  • Use lowercase or snake_case for table and column names
  • Always use WHERE with UPDATE and DELETE unless you mean to affect ALL rows
  • Use parameterized queries or prepared statements to prevent SQL injection
  • Limit query results when you don't need all records (use LIMIT)
  • Index columns used frequently in WHERE, JOIN, ORDER BY
  • Test queries on a small dataset first
  • Use meaningful table and column names
  • Document your database schema with comments

SQL Injection Prevention

SQL Injection is a security vulnerability where malicious SQL is inserted into queries:

  • Use Parameterized Queries / Prepared Statements - separates code from data
  • Validate Input - check data types, lengths, and formats
  • Least Privilege Principle - database users should have only necessary permissions

Real-World SQL Use Cases

  • E-commerce: product search, order history, inventory management
  • Analytics: sales reports, user behavior analysis, business intelligence
  • Web Apps: user authentication, content management, data storage
  • Finance: transaction records, account management, fraud detection
